Constraints
--------
Constraints are rules applied to a table column in my sql to ensure that only valid and accurate data is stored in the database.
Why do we use constraints:-
-----------------------------
1.Maintain data integrity
2.To prevent invalid data entry
3. Avoid duplicate records.
4. ensures relations between tables.
5. make the data base reliable and constraint.
Types of Constraints:-
====================
1. not null :it does not allow null values 
can take duplicates
2.Unique key: it does not allow duplicate values we can take unique values duplicates.
3.primary key: it the combination of unique + not null.: cannot apply more primary keys, it supports for only table level, it acts as a key value pair.
4. Composite key: multiple columns for primary key.
*5. Foreign key: we can merge two tables. a foreign key is a field or collection of fields (collection of fields)in one table that referee to the primary key in another table.
the table containing the foreign key is called child table and the table containing the candidate key is called referenced table or parent table.
6. check constraint: it is like conditional value for ex: if we give a condition between 50 to 100.
7. default constraint: it will give by default values.
================================================================================
What are Referential Actions:-
Referential actions :it defines what happens to child table data when parent table data is updated or deleted .
they are used with foreign key constraints;
1. on delete cascade:
when a parent table row is deleted ,all matching child rows are automatically deleted.
2.on update cascade: when a parent key is updated , all matching child foreign key are updated automatically.
