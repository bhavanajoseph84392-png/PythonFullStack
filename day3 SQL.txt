SQL Identifiers:-
-----------------
Identifiers are the names to identify database objects such as database names, table names,  colum  names, view names, index names, constraints, triggers, or  stored procedures. while we are declaring  identifiers we need to follow some rules and regulations.
1.the only allowed characters to declare identifiers can contain a-z, A-Z or 0-9,_,$.
2.Must begin with letter or underscore(_).
3.We cannot declare identifiers with reserved keywords.
4.Spaces are not considered to declare identifiers.
5.Identifers are case sensitive.
6.Max length to declare identifiers are 15 characters.
SQL:-(Structured Query Language)
--------
It is a standardised programming language specially designed for managing and manipulation relational databases.it provides declarative syntax that allows users to define what data they want retrieve, insert, update, or delete without specifying exactly how database executes these operations.

Types of SQL Commands:-
1.DDL:-data definition language:
=================================
 create: by using these commands to create a new databases tables,views,indexes,trigger,stored procedures.
create database:
-------------------
syntax: create database database-name
example: create database pfs-3;

syntax
------------
create table table-name(
column-name data-type size constraint,-->constraint is optional,
column-name data-type size constraint ,
-------------------------------------
-------------------------------------
);





 alter: by using these alter command to modify the structure of an existing database object.
drop: by using these command we can drop the existing database object or table permanently.
truncate :it is used to delete all rows present in a table.
rename: it used to rename the table names or column names ..etc

2.DML:-(Data Manipulation Lang)
================================
insert: by using these command we can insert some records into a table.
update: by using these command we can update the existing records.
delete: by using this command we can delete the existing table records based on some condition.
lock: it can be used to control access a resource , ensuring that only one transaction. Can be accessed it at a time of preventing conflicts.

3.DCL:-(Data Control Lang)
==================================
grant: this command is used to give specific permission to a user in data base this permission might be include the ability to select,  update or delete as we can create or drop tables
revoke: is used to takeaway granted permissions from user.

4.TCL:-(Transaction Control Lang)
====================================
commit: this command is used to permanently save the changes made during the current transaction .
once committed it changes becomes permanent and visible to other transaction.

rollback: if an error occurs or if transaction needs to be canceled for any reason rollback is used to revert the database
save point: it used to set a point within a transaction who which can later rollback.
5.DQL:-(Data Query Lang)
select: this command is used to retrieve data from one or more tables.
